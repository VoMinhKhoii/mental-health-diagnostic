Dear Team,
I would like you to review  the following points regarding how we might best integrate newly released psychological tests into the platform. Please use the recently released Wechsler Adult Intelligence Scale – Fifth Edition (WAIS-V, 2024) as a working example when thinking through the implementation.
The objective is to design a flexible and user-friendly system that allows clinicians to integrate and use new tests smoothly, regardless of whether they prefer manual scoring or automated scoring through external platforms.
Please also review the three documents attached to this email, as they illustrate the workflows and reporting structure we need the platform to support.
1. Attached Documents
• Spreadsheet with scoring macros
This spreadsheet demonstrates a manual scoring workflow. It includes automated calculations that convert raw scores into scaled scores, percentiles, and composite results, and it automatically generates tables and graphs.
• Auto-scoring report from the Pearson platform (Q-Global format)
This document represents the automated scoring workflow. It is an example of a report generated externally by the Pearson Q-Global system.
• Wechsler Intelligence Test for Children – Sample Interpretative Report
This document contains the full wording, structure, and interpretative sections used to generate a comprehensive psychological report alongside the scores. It includes narrative explanations, interpretations, and structured descriptions of results.
This third document should serve as a reference example for the type of complete interpretative report we ultimately want the platform to generate, particularly for the WAIS-V (2024). The structure and wording can guide the development of automated interpretation sections linked to the score profiles.
2. Two User Pathways the Platform Must Support
• Manual scoring pathway
Users enter raw scores directly into the platform.
The system should then automatically:
Convert raw scores into scaled scores and percentiles
Generate tables and visual graphs
Organize results according to the test’s structure (indexes, subtests, composites)
Produce formatted outputs similar to those generated by the spreadsheet macros
• Automated scoring pathway (external report upload)
Users who score tests through the Pearson Q-Global system should be able to upload their auto-generated report into the platform.
The platform should then:
Extract the relevant data (scores, percentiles, tables, etc.)
Automatically structure the results inside the platform
Associate each score with the appropriate test/subtest and index
Reconstruct tables and visualizations in the platform interface
3. Interpretation and Reporting Layer
Beyond simply importing or calculating scores, the platform should be capable of generating structured interpretationsbased on the score profile.
This should include:
Identification of strengths and weaknesses
Interpretation of index-level patterns
Automatic generation of tables, graphs, and narrative summaries
Conclusions and recommendations
These recommendations could come from:
A structured knowledge base within the platform
Suggestions generated by our AI assistant
At a later stage, I will also provide the descriptions of each subtest, including:
What each test measures
The cognitive or intellectual domains involved
These descriptions should allow the system to support deductive interpretation logic when generating reports.
4. Goal for the Platform
The main goal is to create a system that makes the integration of newly released psychological tests simple and scalable, while remaining intuitive for clinicians who may have different scoring preferences.
Using the WAIS-V (2024) example should help you think through how this process can be generalized for other future tests.
